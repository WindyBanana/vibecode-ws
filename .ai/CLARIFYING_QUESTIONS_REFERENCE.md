# Quick Reference: Clarifying Questions

When user's request is unclear, use these proven question templates.

## Feature Type Questions

### "Add a form"
```
What kind of form?

1. **Contact** - Let people reach you
2. **Sign up** - Create user accounts
3. **Survey** - Collect responses

Which one?
```

### "Add authentication"
```
How should users sign in?

1. **Email + password** - Traditional
2. **Social only** - Google, GitHub
3. **Both** - All options

What do you prefer?
```

### "Add a blog"
```
How to manage posts?

1. **Markdown files** - Simple, no database
2. **Admin panel** - Edit through a form
3. **CMS** - Use external tool

Which fits your needs?
```

### "Add a database"
```
What data needs saving?

1. **User info** - Profiles, settings
2. **Content** - Posts, products
3. **Activity** - Logs, analytics

What are you storing?
```

### "Make it pretty"
```
What style fits your project?

1. **Professional** - Clean, minimal
2. **Fun** - Colorful, playful
3. **Modern** - Bold, trendy

Which vibe?
```

## Scope Questions

### "Add features"
```
What specific feature should I add?

1. **User accounts** - Login/signup
2. **Data storage** - Save information
3. **Third-party service** - Connect to API
4. **Tell me** - What do you need?

Which one?
```

### "Make it better"
```
What specifically needs improvement?

1. **Design** - How it looks
2. **Features** - What it does
3. **Speed** - How fast it loads

What needs work?
```

### "Fix it"
```
What's not working?

1. **Error message** - Tell me what it says
2. **Expected behavior** - What should happen?
3. **Show me** - Can you show a screenshot?

What's the issue?
```

## Technical Clarifications

### "I need an API"
```
What do you mean by API?

1. **Get data** - From weather, maps, etc.
2. **Provide data** - Let apps access your data
3. **Something else** - Explain more

Which one?
```

### "I need a server"
```
What do you need exactly?

1. **Hosting** - Put site online (Vercel)
2. **Database** - Store data (Convex)
3. **Backend** - Run server code

Which fits?
```

### "Add JavaScript"
```
What should JavaScript do?

1. **Interaction** - Buttons, forms, animations
2. **Data** - Fetch/display from API
3. **Something specific** - Tell me more

What behavior do you want?
```

## Data Questions

### "Save user data"
```
What should be saved?

1. **User profiles** - Name, email, settings
2. **User content** - Posts, uploads
3. **User activity** - History, preferences

What data?
```

### "It should remember"
```
Remember for how long?

1. **This session** - Until page refresh
2. **Forever** - Permanent storage
3. **While logged in** - Per user

Which timeframe?
```

## Deployment Questions

### "Make it live"
```
Do you want:

1. **Free link** - Like yourapp.vercel.app
2. **Custom domain** - Like yoursite.com (costs $10-15/year)
3. **Just testing** - Share with a few people first

Which option?
```

### "Deploy it"
```
Ready for these steps?

1. **Push to GitHub** - I'll handle this
2. **Connect Vercel** - You sign up (free, 1 min)
3. **It goes live** - Takes about 5 minutes total

Ready to start?
```

## Priority Questions

### When multiple features needed
```
What's most important first?

1. **[Feature A]** - [Why it matters]
2. **[Feature B]** - [Why it matters]
3. **[Feature C]** - [Why it matters]

Let's start with one. Which?
```

### When timeline unclear
```
What's your timeline?

1. **Quick prototype** - Basic version today
2. **Polished MVP** - Good version this week
3. **Production ready** - Fully tested

Which timeframe?
```

## Budget/Service Questions

### When paid service might be needed
```
This feature works best with [Service].

They have:
- **Free tier** - [What's included]
- **Paid tier** - [What it costs]

Start with free tier?
```

### When free alternatives exist
```
Two options:

1. **Free but limited** - [Service A] (Good for starting)
2. **Paid but full-featured** - [Service B] ($X/month)

Which fits your budget?
```

## User Audience Questions

### "Who will use this?"
```
Who's the audience?

1. **Just you** - Personal use
2. **Small team** - 5-10 people
3. **Public** - Anyone online

Who's it for?
```

### "How many users?"
```
Expected traffic?

1. **Small** - Under 100 users
2. **Medium** - Hundreds of users
3. **Large** - Thousands+

What do you expect?
```

## Content Questions

### "Add content"
```
Where's the content coming from?

1. **You write it** - Manual content
2. **Users create it** - User-generated
3. **From elsewhere** - API/feed

Where's it from?
```

### "How often changes?"
```
How often updates?

1. **Rarely** - Static content (fastest)
2. **Weekly** - Blog-style (balanced)
3. **Constantly** - Real-time (dynamic)

How often?
```

## Verification Questions

### Before building
```
Before I start, is this right?

You want: [Summary in their words]

I'll build:
- [Thing 1]
- [Thing 2]
- [Thing 3]

Match your vision?
```

### After building
```
Done! Here's what you got:
- [Feature 1]
- [Feature 2]
- [Feature 3]

Is this what you wanted?
Need any adjustments?
```

## Default Suggestions

### When user says "I don't know"
```
No worries! For [your situation], I recommend [option].

Because:
- [Simple reason 1]
- [Simple reason 2]

We can change it later. Sound okay?
```

### When user says "whatever you think"
```
I need one input from you to make it perfect:

[Single specific question]

Then I'll handle the rest!
```

## Response Patterns

### ✅ Short and clear
```
What [specific thing]?

1. Option A - One line
2. Option B - One line
3. Option C - One line

Which one?
```

### ❌ Too long
```
There are many approaches we could take here, including...
[Paragraph of technical details]
...which would you prefer?
```

## Remember

- **Max 3-4 options**
- **One line per option**
- **One question at a time**
- **Use plain English**
- **Verify before building**
- **Verify after building**

When in doubt, ask!
